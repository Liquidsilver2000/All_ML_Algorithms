Foundation

What to Choose Depends on 3 Things

Selecting the right algorithm isn't magic — it's a decision based on three key factors that every data scientist considers.

Data Type

The structure and format of your input data determines which algorithms can process it effectively.

  • Structured tables & CSV
  • Text & documents
  • Images & photos
  • Audio & speech
  • Interaction & behavior
  • Graph & network
  • Time series

Goal

What you want to achieve with your model — the prediction task or business objective.

  • Predict values
  • Classify categories
  • Cluster & group
  • Detect anomalies
  • Generate content
  • Recommend items
  • Understand relationships

Business Context

Real-world constraints that affect which model makes sense in production.

  • Accuracy requirements
  • Explainability needs
  • Speed & latency
  • Scale & volume
  • Cost & budget
  • Training data size
  • Maintenance ease

Linear Regression

Structured Data

Best for: Predicting continuous numeric values

Input Data Numeric tables, CSV, structured features
Typical Use Cases Sales prediction, pricing models, demand forecasting, score prediction
Strengths Simple, fast, interpretable, works well with linear relationships
Limitations Struggles with nonlinear complexity, assumes linear relationship
When NOT to use When relationships are highly nonlinear or interactions are complex

XGBoost / LightGBM

Structured Data

Best for: Structured/tabular classification and regression

Input Data Tabular data, numeric & categorical features
Typical Use Cases Churn prediction, fraud detection, credit scoring, ranking
Strengths Powerful, accurate, handles nonlinear relationships, feature importance
Limitations Less interpretable than linear models, can overfit
When NOT to use When model explainability is critical and simple models suffice

K-Means

Structured Data

Best for: Clustering similar observations into groups

Input Data Numeric features, vector representations
Typical Use Cases Customer segmentation, behavior grouping, data preprocessing
Strengths Simple, fast, scales to large datasets, easy to implement
Limitations Requires choosing K, assumes spherical clusters, sensitive to outliers
When NOT to use When cluster shapes are irregular or cluster count is unknown

Isolation Forest

Structured Data

Best for: Anomaly and outlier detection

Input Data Tabular numeric data, feature vectors
Typical Use Cases Fraud detection, unusual transactions, network intrusions
Strengths Efficient for outlier detection, low memory usage, fast training
Limitations May be sensitive to noisy data, struggles with high-dimensional data
When NOT to use When anomalies are subtle or data has complex feature interactions

LSTM

Time Series

Best for: Sequential patterns over time

Input Data Sequential data, time-ordered features
Typical Use Cases Time-series forecasting, sensor streams, long-term sequential dependence
Strengths Captures temporal dependencies, handles long sequences well
Limitations More complex to train, slower than simpler models
When NOT to use For simple forecasting where ARIMA or Prophet would suffice

Prophet

Time Series

Best for: Business time-series forecasting

Input Data Daily/weekly time series with seasonality
Typical Use Cases Sales forecasting, website traffic, daily/weekly seasonality
Strengths Interpretable, easy to use, handles missing data, seasonality
Limitations May underperform for highly complex sequences
When NOT to use When there are complex nonlinear temporal patterns

TimesGPT

Time Series

Best for: Foundation-model style time-series forecasting

Input Data Time-series data, various frequencies
Typical Use Cases Advanced time-series forecasting across domains
Strengths Powerful on time-based data, pre-trained capabilities
Limitations Depends on model availability and scale, may need fine-tuning
When NOT to use When simple statistical methods work or compute is limited

GPT / Llama / Gemini

Text / NLP

Best for: Text generation, reasoning, summarization, conversational AI

Input Data Text, documents, prompts, conversations
Typical Use Cases Chatbots, content writing, summarization, Q&A systems
Strengths Flexible, general-purpose language understanding, few-shot learning
Limitations Costly to run, can hallucinate, needs guardrails and monitoring
When NOT to use When deterministic, low-latency output is needed on a budget

BERT / RoBERTa

Text / NLP

Best for: Text classification and language understanding

Input Data Text documents, sentences, paragraphs
Typical Use Cases Sentiment analysis, spam detection, intent classification, NER
Strengths Strong contextual understanding, transfer learning, fast inference
Limitations Not primarily for long-form generation, fixed input length
When NOT to use When you need to generate long text or creative writing

Vision Transformer

Computer Vision

Best for: Image classification and visual understanding

Input Data Images, photos, visual data
Typical Use Cases Medical imaging, product classification, object recognition
Strengths Modern and high-performing, captures global image context
Limitations Often needs large data, computationally intensive
When NOT to use With small image datasets or limited compute resources

Diffusion Model

Computer Vision

Best for: High-quality image generation

Input Data Text prompts, noise vectors, conditioning inputs
Typical Use Cases Art generation, creative design, synthetic image creation
Strengths High-quality image synthesis, diverse outputs, controllable
Limitations Computationally expensive, slow inference, needs powerful GPUs
When NOT to use When real-time generation is needed or compute budget is limited

Video Transformer

Video AI

Best for: Video understanding and analysis

Input Data Video clips, frame sequences
Typical Use Cases Action recognition, surveillance, event detection
Strengths Understands temporal visual patterns, state-of-the-art accuracy
Limitations Computationally heavy, needs lots of video data
When NOT to use For simple video tasks where CNN-based methods suffice

Whisper / wav2vec

Audio AI

Best for: Speech recognition and audio understanding

Input Data Audio files, speech recordings
Typical Use Cases Transcription, speech analytics, voice AI assistants
Strengths Powerful for audio tasks, multilingual support, robust
Limitations Audio quality affects performance, background noise sensitive
When NOT to use When audio quality is very poor or for non-speech audio tasks

Collaborative Filtering / Two-Tower

Recommendation

Best for: Recommendation systems

Input Data User-item interactions, ratings, clicks
Typical Use Cases E-commerce, streaming platforms, product suggestions
Strengths Personalized recommendations, scales well, effective
Limitations Cold-start problem for new users/items
When NOT to use When you have no interaction history or very sparse data

Reinforcement Learning / PPO

Recommendation

Best for: Decision-making based on rewards

Input Data Environment states, actions, rewards
Typical Use Cases Robotics, dynamic pricing, game agents, policy optimization
Strengths Learns through interaction, adapts to dynamic environments
Limitations Complex training, data/environment intensive, unstable
When NOT to use When you have static data and supervised learning would work

Graph Neural Network

Graph AI

Best for: Connected and relational data

Input Data Graphs, networks, node-edge relationships
Typical Use Cases Fraud rings, social networks, supply chains, molecule prediction
Strengths Models relationships directly, captures network structure
Limitations Complex architecture, harder to scale, specialized setup
When NOT to use When data has no natural graph structure or relationships
